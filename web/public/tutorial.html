Welcome to Lexo

Lexo is a statically-typed programming language designed to teach you the fundamentals of type systems. It uses a Python-like syntax with explicit type annotations — every variable, parameter, and return value has a declared type.

1. Hello World

Use print() to display output. Strings can use double or single quotes. 

print("Hello, World!")
print('Single quotes work too!')

2. Variables & Types

Every variable must declare its type. Lexo has four basic types and four array types:

TypeDescriptionExample
intWhole numbers42, -7
floatDecimal numbers3.14, -0.5
stringText"hello", 'world'
boolBooleantrue, false
int[]Array of ints[1, 2, 3]
float[]Array of floats[1.0, 2.5]
string[]Array of strings["a", "b"]
bool[]Array of bools[true, false]

Declare variables with name: type = value:

name: string = "Alice"
age: int = 25
height: float = 5.7
active: bool = true

print("Name: " + name)
print(age)
print(height)
print(active)
No implicit type coercion. You can't assign a string to an int variable or add an int to a float. This is intentional — Lexo teaches you to think about types explicitly.

4. Control Flow

If / Elif / Else

Conditions must be bool expressions. Use 4-space indentation for blocks (like Python): 

age: int = 15

if age >= 18:
    print("Adult")
elif age >= 13:
    print("Teen")
else:
    print("Child")

While Loops

i: int = 1
while i <= 5:
    print(i)
    i = i + 1

For Loops

The for loop iterates over arrays. The loop variable must include a type annotation: 

fruits: string[] = ["apple", "banana", "cherry"]
for fruit: string in fruits:
    print(fruit)

Break & Continue

Use break to exit a loop early and continue to skip to the next iteration: 

i: int = 0
while true:
    i = i + 1
    if i == 3:
        continue
    if i > 5:
        break
    print(i)
# Prints: 1, 2, 4, 5

5. Functions

Define functions with def. All parameters and the return type must be annotated. Use void for functions that don't return a value. 

def greet(name: string) -> string:
    return "Hello, " + name + "!"

def add(a: int, b: int) -> int:
    return a + b

def sayHi() -> void:
    print("Hi!")

print(greet("World"))
print(add(3, 4))
sayHi()

Recursion

Functions can call themselves:

def factorial(n: int) -> int:
    if n <= 1:
        return 1
    return n * factorial(n - 1)

print(factorial(6))
# Prints: 720

6. Arrays

Arrays hold multiple values of the same type. Access elements with bracket notation (zero-indexed): 

scores: int[] = [90, 85, 92, 78, 95]

# Access by index
print(scores[0])    # 90
print(scores[4])    # 95

# Get the length
print(scores.length())  # 5

# Sum all scores
total: int = 0
for s: int in scores:
    total = total + s
print(total)  # 440

7. String Operations

Strings have built-in methods:

MethodReturnsDescription
.length()intNumber of characters
.upper()stringUppercase copy
.lower()stringLowercase copy
.substring(start, end)stringSlice from start to end
 
msg: string = "Hello, World!"

print(msg.length())        # 13
print(msg.upper())         # HELLO, WORLD!
print(msg.lower())         # hello, world!
print(msg.substring(0, 5)) # Hello

8. Input

Use input() to read text from the user. It always returns a string: 

name: string = input("What is your name? ")
print("Hello, " + name + "!")

9. Comments

Use # for single-line comments. Everything after # on that line is ignored: 

# This is a comment
x: int = 42  # This is also a comment
print(x)

10. Type Errors

One of the key lessons Lexo teaches is that types must match. These programs will produce clear error messages: 

# Type mismatch in assignment
count: int = 10
count = "five"    # Error! Can't assign string to int
# Wrong argument types
def add(a: int, b: int) -> int:
    return a + b

add("hello", "world")  # Error! Expected int, got string
Why does this matter? In real-world programming, type errors caught at compile time save you from bugs that would otherwise only appear when your program runs. Lexo catches these errors before your code executes, just like Go, Java, and TypeScript do.
